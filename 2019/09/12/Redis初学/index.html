Redis初学

数据库 数据库 缓存 面试 2019/09/12

《Redis入门指南第二版》作者：李子骅

1. Redis简述

Redis是一个开源的、高性能的（基于内存）、基于键值对的缓存与存储系统。通过提高多种键值数据类型来适应不同场景下的缓存与存储需求。同时Redis的诸多高层级功能使其可以胜任消息队列、任务队列等不同角色。

2. Redis特性

2.1 存储结构

Redis是Remote Dictionary Server（远程字典服务器）的缩写，他以字典结构存储数据，并允许其他应用通过TCP协议读写。目前为止Redis支持的值数据类型如下：

字符串（String）

散列（Hash）

列表（List）

集合（Set）

有序集合（Sorted Set）

2.2 内存存储与持久化

Redis数据库中的所有数据都存储在内存中，所以Redis相对于其他硬盘存储的数据库在性能上有明显优势。

内存中的数据存在丢失的问题，但Redis提供对持久化的支持，即可以将内存中的数据异步写入到硬盘中，同时不影响继续提供服务。

2.3 功能丰富

Redis可以为每个键设置生存时间（Time To Live, TTL），生存时间到期后键会自动删除，这一功能使得Redis可以用作缓存系统来使用。作为缓存系统，Redis还可以限定数据占用的最大内存空间，当数据达到空间限制后按照一定规则自动淘汰不需要的键。

Redis的列表可以用来实现队列，并且支持阻塞式读取，可以很容易地实现一个高性能的优先级队列。

Redis支持“发布/订阅”的消息模式，可以基于此构建聊天室等系统

3. Redis安装与启动

3.1 Linux安装（centOS、Ubuntu）

# wget http://download.redis.io/redis-stable.tar.gz

# tar xzf redis-stable.tar.gz

# cd redis-stable

# make（root权限下）

# make install

3.2 启动Redis

$ redis-server —port 6380

使用redis-server命令即可启动Redis，默认端口为6379，可以使用--port参数自定义端口

3.3 停止Redis

$ redis-cli SHUTDOWN

强行终止Redis进程可能会使数据丢失，所以使用Redis-cli命令行客户端发送SHUTDOWN命令，SHUTDOWN命令会使Redis先断开所有客户端连接，然后根据配置执行持久化，最后完成退出。

Redis-cli(Redis Command Line Interface)是Redis自带的基于命令行的Redis客户端。

开启命令行

$ redis-cli -h 127.0.0.1 -p 6380

交互

127.0.0.1:6380> PING

返回

PONG

3.4 命令返回值
  1. 状态回复

如返回OK表示设置成功（类似http状态码200）等

  1. 错误回复

命令不存在或格式有错，返回以“ERR”开头的错误提示

  1. 整数回复

Redis没有整数类型，但提供了一些用于整数操作的命令。如递增INCR

redis >INCR foo

(integer) 1

  1. 字符串回复

以双引号包裹，当请求键值不存在时返回空(nil)

  1. 多行字符串回复

每行字符串以一个序号开头

redis> keys *

1) “key1”

2) “key2”

3.5 配置

Redis支持很多配置，支持以配置文件来设置这些选项，在启动时将配置文件的路径作为启动参数传递给redis-server

$ redis-server /path/redis.conf

如果在启动参数中配置了同名配置参数，会覆盖配置文件中的

$ redis-server /path/redis.conf —loglevel warning

除此之外，Redis支持在redis-cli命令行通过CONFIG SET在运行时修改部分配置。

3.6 多数据库

一个Redis实例实际上提供了多个用来存储数据的字典，客户端可以指定将数据存储在哪个数据库中。

redis > SELECT 1

OK

redis [1] >GET foo

(nil)

要注意三点：

  1. Redis不支持自定义数据库名字，每个字典以编号命名

  2. Redis不支持为每个数据库设置不同的访问密码，要么访问全部，要么没有权限

  3. 多个数据库并不是完全隔离的，比如FLUSHALL可以清空一个Redis实例中的所有数据库。

所以同一个Redis实例的数据库更像一种命名空间，而不适宜存储不同应用程序的数据，不同的应用程序应该使用不同的Redis实例存储数据。

4. Redis命令操作

Redis命令不区分大小写

4.1 key命令（关于键的命令）
  1. 获得符合规则的键名列表

KEYS *

判断一个键是否存在（存在返回1，否则返回0）

EXISTS key

  1. 删除键 （返回删除的键的个数）

DEL key

  1. 获得键值的数据类型

TYPE key

4.2 字符串类型命令

字符串类型是Redis中最基本的数据类型，能存储任何形式的字符串，包括二进制数据。一个字符串类型键允许存储最大512MB。字符串类型是其他4中类型的基础。

  1. 赋值与取值

SET key value

GET key

MSET key [key…]

MGET key [key…]

  1. 递增递减数字

INCR key //加1

INCRBY key increment //加increment

DECR key

DECRBY key increment

DECRBYFLOAT key increment

当存储的字符串是整数形式时，Redis提供了一个实用的命令INCR，数字递增并返回递增后的值。当键值不是整数时Redis会提示错误。

包括INCR在内的所有Redis命令都是原子操作（原子操作是最小的执行单位，不会在执行过程中被其他命令插入打断），即当同一时间有多个客户端连接到Redis时，也不会出现竞态条件。

  1. 向尾部追加值

APPEND key value

向字符串末尾追加value，若不存在则将键值设置为value

  1. 获取字符串长度

STRLEN key

4.3 散列（Hash）类型命令

散列类型的键值也是一种字典结构，存储字段与字段值的映射，但字段值只能是字符串（除了散列，Redis的其他数据类型也不支持数据类型嵌套）

散列类型适合存储对象。

  1. 赋值与取值

HSET key field value

HGET key field

HMSET key [field value…]

HMGET key [field value…]

HGETALL key

HKEYS key //只获取字段名

HVALS key //只获取字段值

HLEN key //获取字段数量

HSET命令不区分插入和更新操作，如无，则插入，如已有，则更新。

  1. 当字段不存在时赋值

HSETNX key field value

“NX”表示“if Not eXists”如果字段已经存在，则该命令不执行任何操作；当不存在时，赋值。

  1. 删除字段

HDEL key [field…]

4. 4 列表类型

列表类型可以存储一个有序的字符串列表，内部使用双向链表实现。意味着向两端查询、添加删除元素很快，但通过索引访问元素比较慢。

  1. 从两边添加、删除元素

LPUSH key [value…] //Left PUSH返回增加元素后列表的长度

RPUSH key [value…]

LPOP key //从列表左边弹出一个元素，返回元素值

RPOP key

使用上述命令组合，可以将列表作为栈或队列使用。

  1. 其他命令

LLEN key

获取列表中元素个数

LRANGE key start stop

获得列表的某一片段，起始索引为0，不会删除片段，要注意的是LRANGE截取的片段包括最右边的元素，即[start…stop]，而不是[start…stop)，索引也支持负索引，LRANGE负索引表示从右边开始计数。

LREM key count value

LREM命令会删除列表中前count个值为value的元素，返回值为实际删除的个数。

LTRIM key start stop

只保留指定片段

4.5 集合类型命令

集合中数据具有无序性、唯一性。集合类型在Redis内部是使用值为空的散列表实现的。

方便的在于多个集合类型键之间可以进行并集、交集和差集运算。

  1. 增加/删除元素

SADD key [number…] //若加入的键值已存在，则忽略这个元素，返回成功加入的元素数量

SREM key [number…]

  1. 获取集合所有元素

SMEMBERS key

  1. 判断元素是否在集合中

SISMENBER key member

member存在于key中，返回1，否则返回0

  1. 集合间运算

SDIFF [key…]

差集运算，如SDIFF setA setB表示setA - setB，所有属于A但不属于B的元素

SINTER [key…]

交集运算，所有属于A且属于B的元素

SUNION [key…]

并集计算，所有属于A或属于B的元素

4.6 有序集合类型命令

与集合类型的区别就是有序。在集合类型的基础上为集合中的每个元素都关联了一个分数（score），按分数排序。使用散列表和跳跃表实现。

  1. 增加元素

ZADD key [score member…]

加入一个元素和该元素的分数，如果已存在则会刷新分数。返回值是成功新加入的元素个数（不包括分数的刷新）

分数也支持双精度浮点数，同时+inf表示正无穷，-inf表示负无穷

  1. 获取元素

ZSCORE key member //获得某元素的分数

ZRANGE key start stop [WITHSCORES] //返回按元素分数从小到大的顺序，start到stop之间的所有元素。

ZREVRANGE key start stop [WITHSCORES] //从大到小…

ZRANGEBYSCORE key min max [WITHSCORES] //按元素从小到大返回分数在min至max之间（包含）的元素

  1. 增加某个元素的分数

ZINCRBY key increment member
